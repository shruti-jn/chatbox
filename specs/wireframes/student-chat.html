Student Chat

Screen: student-chat | Persona: PER-001 (Sam the Student) | Journeys: JRN-001, JRN-002

Information Hierarchy

1
Chat Messages + App Cards (center column) — The conversation thread with embedded app cards is the focal point. This is where learning happens.
2
Chat Input Bar (bottom) — Persistent text input with suggestion chips. Always visible, always accessible. This is the primary action surface.
3
Top Bar (top) — Classroom context, student identity. Minimal chrome. Class name + teacher name orient the student.

Focal Point

FOCAL The most recent message or app card owns the first 3 seconds. On load, scroll to the latest interaction. If an app card is active, it takes visual priority over text messages.

Above-the-fold working surface: The student can read the last 2-3 messages and type a response without scrolling. The chat input is always pinned to the bottom.

Banned Dominance Patterns

  • No hero banner or welcome splash that pushes content below the fold
  • No sidebar navigation consuming horizontal space — chat is full-width (centered, max 800px)
  • No app catalog grid on the chat screen — apps appear only when invoked in conversation
  • Empty state must not be a large illustration dominating the viewport — use a simple prompt

Interactive States

CLICK Suggestion chips: Clicking a chip inserts its text into the input and sends immediately. Chip disappears after use.

CLICK App card: Clicking inside the app card's iframe sends events to the app. The card can expand/collapse. A minimize button collapses the app card to a summary row.

HOVER Messages: Hover reveals a timestamp and a "copy" icon on AI messages.

FOCUS Chat input: Border color changes to primary. Placeholder text: "Ask me anything..." for grades 6-12, "What do you want to learn?" for K-5.

STREAMING AI response: Tokens stream in word-by-word. A typing indicator (three dots) appears while waiting for first token. Scroll auto-follows the streaming response.

API Integration Annotations

API Chat messages zone: GET /api/v1/sessions/{session_id}/messages — loads conversation history on page load. Paginated (newest first, load more on scroll up).

API Send message: POST /api/v1/sessions/{session_id}/messages — sends student message. Returns streaming AI response via WebSocket /ws/chat/{session_id}.

API App card rendering: GET /api/v1/apps/{app_id}/render — returns iframe URL and initial state. App state updates flow via postMessage bridge.

API Suggestion chips: Returned as part of the AI streaming response metadata. No separate endpoint.

API Session context: GET /api/v1/sessions/{session_id} — classroom name, teacher name, grade band, enabled apps.

Responsive Behavior

MOBILE 375px Chat messages go full-width with reduced padding. App cards render at full viewport width. Suggestion chips wrap to 2 rows max. Top bar collapses to logo + hamburger menu.

TABLET 768px Same as desktop but chat column max-width reduces to 640px. App cards maintain aspect ratio.

DESKTOP 1280px Chat column centered at max-width 800px. Generous whitespace on sides. App cards render up to 720px wide.

Persona Fit

PERSONA This screen serves K-12 students (ages 5-18). The layout is intentionally simple — one conversation thread, one input, minimal navigation. Younger students (K-5) see larger text, more suggestion chips, and simpler language. Older students (6-12) see standard text sizes and can type freely. The chat-first paradigm matches how students already communicate (messaging apps). Apps appearing inline in conversation feels natural, not imposed.
