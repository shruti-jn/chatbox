Teacher Dashboard (Mission Control)

Screen: teacher-dashboard | Persona: PER-002 (Ms. Torres the Teacher) | Journeys: JRN-003, JRN-004

Information Hierarchy

1
Safety Alerts Banner (top, conditional) — Red/amber alert bar when safety flags are active. This MUST be the first thing the teacher sees. Shows count, severity, and quick-action buttons. Auto-appears via WebSocket push.
2
Student Conversation Grid (center) — Card grid showing all active student conversations. Each card: student name, last message preview, active app indicator, engagement status (active/idle/offline). This is the primary monitoring surface.
3
Classroom Summary Bar (below top bar) — At-a-glance stats: X students online, Y active conversations, Z app sessions, teacher whisper queue count.
4
Top Bar + Navigation — Classroom selector, config link, analytics link. Teacher name and period.

Focal Point

FOCAL Safety alerts (if any) take absolute priority. If no alerts, the student grid is the focal point. The teacher should be able to scan 20-35 student cards and identify who needs attention within 5 seconds.

Above-the-fold working surface: The teacher can see the summary stats and at least 8-12 student cards without scrolling. Safety alerts are above the grid, never below.

Banned Dominance Patterns

  • No welcome message or onboarding wizard dominating the dashboard after first use
  • No analytics charts on the main dashboard — analytics has its own page
  • No large empty state illustration when some students are online but not chatting
  • Student cards must not be so large that fewer than 6 are visible above the fold

Interactive States

CLICK Student card: Opens a slide-out panel showing the full conversation transcript. Teacher can read all messages and app interactions. Panel has a "whisper" input to inject context into the AI.

CLICK Safety alert: Clicking an alert opens the student's conversation scrolled to the flagged message. Alert badge shows severity (critical = red, warning = amber).

CLICK Whisper button: Opens a text input. Teacher types a prompt that gets injected into the AI's system context for that student. Student does not see the whisper. AI incorporates it into its next response.

HOVER Student card: Subtle elevation increase. Shows "View conversation" overlay text.

REAL-TIME Student status indicators: Green dot = active (sent message in last 2 min). Yellow = idle (2-10 min). Gray = offline. Updates via WebSocket.

REAL-TIME Message preview: Last message on each card updates in real time as students chat. Truncated to 2 lines.

API Integration Annotations

API Student grid: GET /api/v1/classrooms/{classroom_id}/students/activity — returns list of students with status, last message, active app. Polled every 10s OR pushed via WebSocket /ws/teacher/{classroom_id}.

API Safety alerts: GET /api/v1/classrooms/{classroom_id}/safety-alerts — returns unresolved alerts. Also pushed in real-time via WebSocket. Critical alerts trigger browser notification.

API Student conversation (slide-out): GET /api/v1/sessions/{session_id}/messages — full transcript for selected student.

API Teacher whisper: POST /api/v1/sessions/{session_id}/whisper — sends teacher context injection. Body: { "content": "Guide student toward photosynthesis equation" }.

API Summary stats: GET /api/v1/classrooms/{classroom_id}/stats — active count, session count, app session count.

API Resolve alert: PATCH /api/v1/safety-alerts/{alert_id} — marks alert as reviewed/resolved.

Responsive Behavior

MOBILE 375px Student cards stack single-column. Conversation slide-out becomes full-screen overlay. Summary stats collapse to 2 key metrics (students online, alerts). Teachers primarily use this view on tablet or desktop — mobile is emergency-only.

TABLET 768px Student grid: 2 columns. Slide-out panel takes 60% width.

DESKTOP 1280px Student grid: 3-4 columns. Slide-out panel takes 400px on right side. Grid compresses to accommodate panel.

Persona Fit

PERSONA This screen serves teachers managing 20-35 students simultaneously during a 30-50 minute class period. The design prioritizes rapid scanning — card-based grid layout lets teachers identify who is engaged, who is struggling, and who triggered a safety alert. The whisper feature is the key differentiator: teachers can shape AI behavior per-student without interrupting the student's flow. The dashboard must feel like an air-traffic-control view, not a report. Real-time updates are essential — stale data during a class period is unacceptable.
