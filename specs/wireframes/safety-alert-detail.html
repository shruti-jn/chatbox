Safety Alert Detail

Screen: safety-alert-detail | FR: FR-MON-002 | Persona: PER-002 (Ms. Torres the Teacher) | Journeys: JRN-003

Layout Zones

1
Alert Header Bar — Severity badge (CRITICAL / WARNING / INFO), student name, timestamp of detection, and alert category (e.g., "Suicide/Self-Harm", "Violence", "Profanity"). Red background for critical, amber for warning.
2
Flagged Message in Context — Full conversation transcript scrolled to the flagged message. The flagged message is highlighted with a colored left border and a "Flagged" badge. Surrounding messages are shown at reduced opacity to provide context.
3
Action Panel (right or bottom) — Action buttons: "Send Direct Message", "Send Whisper to AI", "Pause AI for Student", "Resolve — No Action Needed", "Escalate to Counselor". Each button opens a confirmation or input step.
4
Alert Metadata Footer — Alert ID, classifier confidence score, detection latency, prior alert count for this student in this session.

Trigger Conditions

TRIGGER WebSocket push: Teacher receives a safety_alert event on WebSocket /ws/teacher/{classroom_id}. The event includes alert_id, conversation_id, severity, and category. The Mission Control dashboard auto-pins the alert and clicking it navigates here.

TRIGGER Click from alert banner: Teacher clicks an alert row on the teacher-dashboard. The alert_id and conversation_id are passed as route params.

TRIGGER Email link (critical only): For severity=critical, an email is sent to the teacher. Clicking the link deep-links to this screen with pre-authenticated token.

API Endpoints

API Load alert detail: GET /api/v1/safety-alerts/{alert_id} — Returns alert metadata: severity, category, flagged_message_id, conversation_id, classifier_confidence, detected_at. Called on screen mount.

API Load conversation transcript: GET /api/v1/conversations/{conversation_id}/messages — Full message history. Cursor-positioned to the flagged message using ?anchor=message_id query param. Returns surrounding context (20 messages before and after).

API Resolve alert: PATCH /api/v1/safety-alerts/{alert_id} — Body: { "status": "resolved", "resolution_note": "false positive" }. Marks alert as reviewed, removes from active alert queue.

API WebSocket (teacher): WebSocket /ws/teacher/{classroom_id} — Stays open. New alerts push in real time while reviewing this one. Badge count in header updates.

State Transitions (FSM-006: Teacher Intervention Flow)

FSM Entry state: alert_triggered → Teacher clicks REVIEW_ALERT event → transitions to reviewing state.

FSM From reviewing, exit paths:

  • SEND_MESSAGE event → intervened (teacher sends direct message)
  • PAUSE_AI event → intervened (teacher pauses AI)
  • SEND_WHISPER event → intervened (teacher whispers to AI)
  • DISMISS event → monitoring (false positive, no action taken)
  • CONTACT_COUNSELOR event → intervened (escalation logged)

FSM Concurrent state: The student's conversation_lifecycle FSM (FSM-001) is not affected by the teacher viewing this screen — the student's session continues while the teacher reviews.

User Exit Paths

  • Resolve — No Action: Marks alert resolved → returns to teacher-dashboard. Alert removed from active queue.
  • Send Direct Message: Opens teacher-direct-message screen overlay. On submit, SEND_MESSAGE event fires → intervened → auto-return to dashboard after confirmation.
  • Send Whisper: Opens teacher-whisper screen overlay. On submit, SEND_WHISPER event fires → intervened → return to dashboard.
  • Pause AI: Opens teacher-pause-ai confirmation overlay. On confirm → intervened → return to dashboard.
  • Escalate to Counselor: Logs escalation event, shows contact info card, returns to dashboard after teacher acknowledges.
  • Back arrow / breadcrumb: Navigates back to teacher-dashboard without resolving alert. Alert remains active in queue.
  • Click another alert (banner): Replaces current alert detail view with new alert. Confirmation modal if current alert not yet resolved.

Error Recovery

ERROR Alert not found (404): Alert may have been resolved by another device. Show "This alert has already been resolved" with a link back to the dashboard.

ERROR Conversation load failure: If GET /conversations/{id}/messages fails, show the alert metadata and action buttons without the transcript. Teacher can still take action. Message: "Conversation transcript unavailable — you can still take action from the panel on the right."

ERROR PATCH resolve failure: Show inline error "Could not mark alert as resolved. Try again." Retry button. Alert remains in active queue.

ERROR WebSocket disconnect while reviewing: Show a reconnecting indicator in the header bar. Buffered alerts are delivered on reconnect. The current review screen is unaffected.

Accessibility Notes

  • Alert severity badge uses both color and text label (not color alone). WCAG 1.4.1.
  • Flagged message is announced via aria-live="assertive" when the screen loads. Screen reader reads: "[Student name]'s message flagged as [category] — [severity]."
  • Action buttons are grouped in a role="group" with aria-label="Intervention actions".
  • Tab order: Alert header → Flagged message → Action panel → Footer metadata.
  • Keyboard shortcut: R to resolve, M to open direct message, P to pause AI. Shortcuts displayed in a tooltip on first visit.
  • Focus is trapped in any overlay (whisper/direct message/pause) until dismissed.
