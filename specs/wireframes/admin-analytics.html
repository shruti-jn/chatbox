Admin Analytics Dashboard

Screen: admin-analytics | Persona: PER-003 (Dr. Patel the District Admin) | Journey: JRN-005

Information Hierarchy

1
Top Stats Row (top) — Four headline metrics: active students, chat sessions, app sessions, avg duration. These are the numbers the admin reports to the school board.
2
Daily Active Students Chart (center) — Time series showing adoption trend. The most important story: is usage growing? This chart answers the board's first question.
3
Two-Column Detail (bottom) — Left: Top Apps by Usage table. Right: Grade Band Distribution bar chart. Supporting detail that explains the headline numbers.
4
Filter Row (below header) — Date range, school, grade band dropdowns. Enables drill-down by school or grade for board presentations.

Focal Point

FOCAL The "Active Students" stat card and the Daily Active Students chart together tell the adoption story. The stat gives the number, the chart gives the trend.

Above-the-fold: Stats row + the daily active students chart must be fully visible without scrolling at 1280px.

Banned Dominance Patterns

  • No "welcome to analytics" or onboarding banner — admins arrive here intentionally
  • No more than 1 line chart above the fold — multiple charts create cognitive overload
  • No pie charts — horizontal bars are more scannable for distribution data
  • Filter row must not push stats below the fold

Interactive States

FILTER Date range dropdown: Changes all data on page. Shows loading skeleton for 200-500ms during data fetch. Default: "Last 30 days".

FILTER School dropdown: Filters all metrics to selected school. "All Schools" is default.

FILTER Grade band dropdown: Filters to selected grade band. Affects all panels.

HOVER Chart bars: Tooltip showing exact date and count ("Mar 15: 142 students"). Bar highlights on hover.

HOVER Grade band bars: Tooltip with exact student count and percentage.

CLICK App name in top apps table: Links to that app's detail page with per-app analytics.

API Integration Annotations

API Summary stats: GET /api/v1/admin/analytics/summary?district_id={id}&date_range=30d&school=all&grade=all — returns active students, session counts, avg duration.

API Daily active students: GET /api/v1/admin/analytics/daily-active?district_id={id}&date_range=30d — returns array of { date, count } for chart rendering.

API Top apps: GET /api/v1/admin/analytics/top-apps?district_id={id}&date_range=30d&limit=5 — returns app name, sessions, avg duration, completion rate.

API Grade distribution: GET /api/v1/admin/analytics/grade-distribution?district_id={id}&date_range=30d — returns grade band percentages.

Responsive Behavior

MOBILE 375px Stats row wraps to 2x2 grid. Chart reduces height to 180px. Two-column section stacks to single column. Filter dropdowns stack vertically. Admin analytics is a desktop-primary screen — mobile is for quick checks during board meetings.

DESKTOP 1280px Full layout as designed. Stats row horizontal, chart full-width, two-column below.

Persona Fit

PERSONA District administrators use analytics to justify procurement renewal (6+ month cycles), report to the school board, and identify adoption gaps across schools. The dashboard is optimized for storytelling: "2,847 students are actively using ChatBridge, up 23% from last month, with strongest adoption in grades 6-8." The filter-by-school capability lets admins identify underperforming schools for targeted support. Data is presented in board-presentation-ready format — the admin can screenshot this page for their quarterly report.
