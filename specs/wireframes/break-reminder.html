Break Reminder & AI Disclosure

Screen: break-reminder | FR: FR-SAFE-010 | Persona: PER-001 (Sam the Student) | Journeys: JRN-001

Layout Zones

1
AI Disclosure Banner (persistent, top of every session) — Shown at the start of every session and always visible in the top bar via a small "i" info icon. Text: "You're chatting with an AI, not a real person. Your teacher can see this conversation." Per EU AI Act Article 50 and California SB 243. The disclosure must be conspicuous, not buried.
2
Break Reminder Card (inline, in chat thread) — Appears as a special chat card after 60 minutes of continuous use. Card style: friendly, non-alarming. Illustration: a simple eye/rest icon. Message: "You've been learning for an hour! Time to give your eyes a rest. Take a 5-minute break before continuing." Two options: "Take a Break" (pauses chat input, starts 5-min countdown) and "Continue Anyway" (dismisses card with a logged acknowledgment).
3
Break Countdown (post "Take a Break") — If student takes the break: chat input is disabled. A countdown timer shows "Break: 4:32 remaining." When timer ends: input re-enables, a gentle "Welcome back!" message appears. Student can click "I'm back early" to return before timer ends.

Trigger Conditions

TRIGGER (AI Disclosure) Session start: Every time a student opens the chat interface (new session or returning). The disclosure appears for 3 seconds as a top banner, then collapses to an "i" icon in the navigation bar. On first-ever login, a modal requires explicit acknowledgment ("Got it!") before the student can type.

TRIGGER (Break Reminder) 60 minutes of continuous active use: Timer starts on first message of the session. Resets if student is idle for more than 10 minutes. "Active use" = at least one message sent in the last 10 minutes. Per AC-SAFE-010-01.

TRIGGER (Break Reminder) Subsequent reminders: If student clicks "Continue Anyway", reminder appears again after another 30 minutes of active use. If student takes the break, timer resets to 60 minutes after break completes.

API Endpoints

API Session usage time: GET /api/v1/sessions/{session_id} — Includes active_duration_minutes computed field. Used on page load to determine if break reminder should appear immediately (e.g., student had 55 minutes of use and refreshed).

API Log break reminder event: POST /api/v1/sessions/{session_id}/events — Body: { "event": "break_reminder_shown" | "break_taken" | "break_dismissed" }. Fire-and-forget. Used in analytics and compliance audit trail.

API Log AI disclosure acknowledgment (first login only): POST /api/v1/users/{user_id}/disclosure-acknowledged — Records that student has seen and acknowledged the AI disclosure. Only called once per user lifetime. Required for California SB 243 audit.

API No endpoint for break countdown: Countdown is client-side only (JS timer). If page refreshes during break, GET /sessions/{id} returns break state and remaining time.

State Transitions

FSM Student FSM-001: Break reminder is orthogonal to the conversation state machine. The conversation remains in active state during a break. The chat input is disabled on the client side only. No server-side state change.

TIMER Client-side state machine (break flow):

  • tracking → 60 min elapsed → reminder_shown
  • reminder_shown + "Take a Break" → on_break (5-min timer, input disabled)
  • reminder_shown + "Continue Anyway" → tracking (reset to 30-min interval)
  • on_break + timer expires → tracking (reset to 60-min interval)
  • on_break + "I'm back early" → tracking (reset to 60-min interval)

User Exit Paths

  • "Take a Break": Disables chat input, shows 5-min countdown. When timer ends, input re-enables. Break event logged.
  • "Continue Anyway": Dismisses reminder card. Reminder re-appears in 30 minutes. Dismissal logged for compliance audit.
  • "I'm back early" (during break): Ends break early, re-enables input, resets 60-min timer.
  • Close browser/tab (during break): Break state is lost — next session starts fresh with a new 60-min timer.
  • Disclosure "i" icon (click): Opens a small popover: "You're talking with an AI assistant. Your teacher [Name] can see this conversation. You are not talking to a real person." Closes on click-outside or Escape.

Error Recovery

ERROR Session GET fails on page load: Client defaults to showing break reminder immediately if usage time is unknown (conservative approach — better to remind than to miss a required reminder).

ERROR Break event log POST fails: Silently swallowed. Never retried in a way that blocks the UI. Break reminder UX is unaffected by log failure.

ERROR Disclosure acknowledgment POST fails: Retried silently 3 times. If all fail, the disclosure modal re-shows on next login (conservative — keeps showing until confirmed). The student can still use the platform in the meantime (non-blocking after first display).

Accessibility Notes

  • AI Disclosure banner uses role="status" and aria-live="polite" on session start. Screen reader announces: "Notice: You are chatting with an AI, not a real person."
  • Break reminder card uses role="alert" (not assertive — it's not urgent). Announced once when it appears.
  • Countdown timer uses aria-live="polite" and updates every minute (not every second — to avoid excessive announcements). Reads: "Break: 4 minutes remaining."
  • Chat input disabled state: aria-disabled="true" and aria-label="Chat is paused during your break. 4 minutes remaining."
  • Disclosure modal (first login): Focus is trapped inside. "Got it" is the only button. Keyboard accessible. Tab cycles through disclosure text and button.
