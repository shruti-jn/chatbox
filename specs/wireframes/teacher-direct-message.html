Teacher Direct Message

Screen: teacher-direct-message | FR: FR-MON-003 | Persona: PER-002 (Ms. Torres the Teacher) | Journeys: JRN-003

Layout Zones

1
Compose Panel (modal or slide-out) — Target student name and profile chip at top. Message text area (no character limit enforced but placeholder: "Write a message to [Student]..."). Suggested quick messages (optional chips): "Great work!", "Please re-read the instructions", "Let me know if you need help". Send button. Cancel button.
2
Student Conversation Preview (context pane) — Last 5 messages of the student's conversation shown read-only. Helps teacher see what the student last said before composing their message.
3
Message Preview (live) — A live preview of how the message will appear in the student's chat: the teacher's icon, "Ms. Torres" label, and message text, styled with the distinct teacher-message color (distinct from AI messages). Labeled "Student will see this."

Trigger Conditions

TRIGGER From teacher-dashboard: Teacher clicks "Message" button on a student card's hover action bar or the slide-out conversation panel.

TRIGGER From safety-alert-detail: Teacher selects "Send Direct Message" as intervention. The compose panel opens with conversation_id pre-filled.

TRIGGER From teacher-pause-ai: Teacher checked "Also send a direct message now" during the pause flow. Compose panel opens after pause is confirmed.

TRIGGER While AI is paused: If student's AI is paused, the "Message" action is surfaced more prominently on the student card (since the teacher is the only one who can communicate with the student now).

API Endpoints

API Load conversation context: GET /api/v1/conversations/{conversation_id}/messages?limit=5&order=desc — Last 5 messages for context preview. Called on panel open.

API Send direct message: POST /api/v1/conversations/{conversation_id}/teacher-message — Body: { "content": "..." }. The message is stored with type="teacher" and pushed via WebSocket to the student's chat thread with distinct styling (teacher icon + name + different color). Returns { "message_id": "uuid", "delivered_at": "ISO8601" }.

API WebSocket (student side): Student receives teacher_message event on /ws/chat/{session_id}. The message appears immediately in the chat thread with a teacher icon and distinct color, no page refresh needed.

State Transitions

FSM Teacher FSM-006: SEND_MESSAGE event from reviewing → intervened. log_intervention action records the direct message in the audit trail (author, timestamp, content).

FSM Student FSM-001 (if not paused): No state change to the conversation FSM. The teacher message is stored and delivered; it does not affect the conversation_lifecycle state machine. The student can respond with their own message.

FSM Student FSM-001 (if paused): Conversation is in paused state. TEACHER_MESSAGE event is valid from paused state → remains in paused. deliver_teacher_message action fires.

User Exit Paths

  • Send message: POST fires → success toast "Message delivered to [Student]" → panel closes → returns to invoking context (dashboard or alert detail).
  • Cancel button: Panel closes. No API call. Returns to invoking context. If text was typed, "Discard message?" confirmation shown.
  • Escape key: Same as Cancel.
  • Quick message chip: Chip text populates the text area. Teacher can edit or send immediately. Clicking a chip does not auto-send.

Error Recovery

ERROR POST fails (5xx / network): Inline error below compose area: "Message could not be delivered. Try again." Message text is preserved. Retry button. Student did not receive the message.

ERROR POST fails (403 — not teacher's classroom): Show "You do not have permission to message this student." This should not occur in normal flow (teacher can only see their own classroom's students).

ERROR POST fails (404 — conversation ended): "This student's session has ended. Message not delivered." Panel closes. Student card updates to show offline status.

ERROR Student WebSocket offline: Message is stored server-side. When student reconnects, the teacher message appears in their chat history. The POST still returns 200 with a delivery_pending flag. Teacher sees "Message queued — will deliver when student reconnects."

Accessibility Notes

  • Panel is role="dialog" with aria-modal="true". Focus trapped inside while open.
  • Text area has aria-label="Message to [Student Name]" and aria-required="true".
  • Quick message chips use role="button" with aria-label="Use quick message: [chip text]".
  • Message preview section has aria-label="Preview: how student will see this message" and aria-live="polite" so screen readers announce changes as teacher types.
  • Send button is disabled and aria-disabled="true" when text area is empty. Screen reader announces: "Send — disabled until you type a message."
