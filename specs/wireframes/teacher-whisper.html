Teacher Whisper

Screen: teacher-whisper | FR: FR-MON-005 | Persona: PER-002 (Ms. Torres the Teacher) | Journeys: JRN-003

Layout Zones

1
Whisper Compose Panel (modal or slide-out) — Target student name and current topic at top. Text area for whisper (max 500 chars, char counter shown). Placeholder text: "e.g., Guide the student toward the photosynthesis equation without giving the answer." Expiry selector: "Active for next 1 / 3 / 5 AI responses" (dropdown). Send button. Cancel button.
2
Student Conversation Preview (context pane) — Last 5 messages of the student's conversation shown read-only at reduced size. Helps teacher compose a relevant whisper. Not interactive.
3
Active Whispers List (below compose) — Any whispers currently active for this student. Each row: whisper text preview, remaining responses, "Cancel whisper" button. Empty state: "No active whispers for this student."

Trigger Conditions

TRIGGER From teacher-dashboard: Teacher clicks the "Whisper" button on a student card (visible on hover). The conversation_id is passed as context.

TRIGGER From safety-alert-detail: Teacher selects "Send Whisper" as their intervention action. Pre-populates with the alert's conversation_id.

TRIGGER From student conversation slide-out: Teacher is reviewing a transcript and clicks the "Whisper" icon in the action bar.

API Endpoints

API Load active whispers: GET /api/v1/conversations/{conversation_id}/whispers?active=true — Returns whispers currently in the AI's context for this student. Used to populate the active whispers list. Called on panel open.

API Load conversation context: GET /api/v1/conversations/{conversation_id}/messages?limit=5&order=desc — Last 5 messages for the context preview pane.

API Send whisper: POST /api/v1/conversations/{conversation_id}/whisper — Body: { "whisper_text": "...", "expires_after_minutes": 15 }. Returns { "whisper_id": "uuid" }. The whisper is included in the AI's context assembly for the next N responses. Student never sees it.

API Cancel whisper: DELETE /api/v1/conversations/{conversation_id}/whispers/{whisper_id} — Removes a whisper from active context immediately.

State Transitions (FSM-006: Teacher Intervention Flow)

FSM Entry: Teacher is in reviewing state (or monitoring state if whisper initiated from dashboard).

FSM On send: SEND_WHISPER event fires → transition to intervened. The log_intervention action records the whisper action in the audit trail.

FSM Student conversation (FSM-001): No immediate state change to the student's session. The whisper is included passively in the next AI context assembly. The student's conversation remains in active state.

FSM On cancel (discard): No event fired. Teacher returns to prior state (monitoring or reviewing) without state change.

User Exit Paths

  • Send Whisper (submit): POST fires → success toast "Whisper sent — AI will incorporate this in its next response" → panel closes → returns to invoking screen (dashboard or alert detail).
  • Cancel button: Closes panel without sending. Returns to invoking screen. No API call.
  • Escape key: Same as Cancel. If text has been typed, shows "Discard whisper?" confirmation before closing.
  • Click outside panel (if modal): Same as Escape — discard confirmation if text present.
  • Cancel active whisper (from list): DELETE fires → whisper row removed from list. Teacher remains on this panel to compose a new one or close.

Error Recovery

ERROR POST fails (network/5xx): Inline error below the text area: "Whisper could not be sent. Try again." Retry button. Whisper text is preserved in the form.

ERROR POST fails (400 — text too long): Client-side validation prevents submission above 500 chars. If server returns 400, show "Whisper text exceeds maximum length (500 characters)."

ERROR GET active whispers fails: Show "Could not load active whispers" with a retry link. The compose area remains functional so the teacher can still send a new whisper.

ERROR DELETE cancel-whisper fails: Show inline error on the whisper row: "Could not cancel — try again." Whisper row remains visible.

ERROR Student session ended before send: Server returns 404 on the conversation. Show "This student's session has ended. Whisper not delivered." Offer to return to dashboard.

Accessibility Notes

  • Panel is a modal dialog with role="dialog", aria-modal="true", and aria-labelledby pointing to the panel heading.
  • Focus is trapped inside the modal while open. On close, focus returns to the element that opened it (whisper button on student card).
  • Character counter is announced via aria-live="polite" as teacher types. Reads: "340 of 500 characters used."
  • Send button is disabled and aria-disabled="true" when text area is empty.
  • Active whispers list uses role="list". Each cancel button has aria-label="Cancel whisper: [first 30 chars of whisper text]".
