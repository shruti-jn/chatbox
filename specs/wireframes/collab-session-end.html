Collaborative Session End

Screen: collab-session-end | FR: FR-COLLAB-005 | Personas: PER-001 (Student), PER-002 (Teacher) | Journeys: JRN-002

Layout Zones

1
Session Summary Card (in each participant's chat thread) — Appears as an AI-generated message card at the end of the collaborative session. Title: "[App Name] Session Complete" with a checkmark icon. Content (AI-generated per participant): acknowledgment of what was accomplished, key moments ("You made 12 moves and reached checkmate in 23 moves!"), and a learning reflection ("What strategy worked best for you?"). Each participant receives a personalized summary card.
2
Session Stats Row (inside summary card) — Duration, number of actions taken by this participant, participants in session, final state summary (e.g., "Winner: You!" or "Score: 8/10 flashcards"). Stats are derived from the session's final state snapshot. They are factual, not generated.
3
Return to Chat Prompt (below summary card) — The AI continues the individual conversation with a follow-up prompt based on the session. Example: "Would you like to keep practicing? I can set up another chess game or try a different activity." Chat input re-activates. The collaborative session WebSocket is closed; only the individual chat WebSocket remains.
4
LTI Grade Submission Notification (conditional) — If the classroom uses LTI grading and the app is configured for grade submission, a small badge appears on the summary card: "Grade submitted to [LMS name]." Teacher sees this in their Mission Control analytics view for the session. Students see a simplified version: "Your progress has been recorded." No grade values are shown to students.

Trigger Conditions

TRIGGER Activity completes naturally: App signals activity completion via CBP (e.g., checkmate detected, all flashcards reviewed, quiz finished). ACTIVITY_COMPLETE event fires on FSM-003. Session transitions to ending state.

TRIGGER Teacher force-closes session: Teacher clicks "End Session" on the collaborative session card in Mission Control or the student's conversation slide-out. TEACHER_END event fires. All participants receive immediate notification.

TRIGGER 30-minute inactivity timeout: No PLAYER_ACTION events for 30 minutes. Server fires the timeout transition from active to ending. Per FSM-003 timeout_transitions.

TRIGGER All participants disconnected: ALL_DISCONNECTED event → ending. Session summary is generated and delivered when participants return.

SEQUENCE Ending → Ended flow: On entering ending, generate_summary and persist_final_state actions fire. When AI summary is ready, SUMMARY_READY event fires → ended. deliver_summary action broadcasts summary card to all participants via WebSocket.

API Endpoints

API End session (teacher or creator): POST /api/v1/collaborative-sessions/{session_id}/end — Triggers TEACHER_END event. Response: { "status": "ending", "summary_eta_seconds": 10 }. Summary is generated asynchronously.

API Summary generation (internal): POST /api/v1/ai/completions — Internal call made by the server to generate a per-participant summary. Not client-initiated. Prompt includes session state, action log, and participant interaction counts.

API WebSocket summary delivery: WebSocket /ws/collab/{session_id} receives SUMMARY_READY event: { "type": "session_summary", "summary": "...", "stats": { ... } }. This is the last message on the collaboration WebSocket before it closes. After delivery, the server closes the WebSocket connection.

API LTI grade submission (conditional): POST /api/v1/lti/grades/{app_id} — Internal call during ended entry action if LTI grading is configured. Not client-triggered. On success: LTI grade badge appears on summary card via WebSocket update to individual chat /ws/chat/{session_id}.

API Individual chat WebSocket resumes: After collab session ends, the student's individual chat /ws/chat/{conversation_id} becomes the active connection. The AI follow-up message is delivered via this channel.

State Transitions (FSM-003: Collaborative Session Lifecycle)

FSM Active → Ending: ACTIVITY_COMPLETE, TEACHER_END, ALL_DISCONNECTED, or 30-min timeout → ending. Entry actions: generate_summary (async AI call) and persist_final_state (sync DB write).

FSM Ending → Ended: SUMMARY_READY event → ended. deliver_summary action broadcasts to all participants. Entry actions on ended: close_connections (closes all session WebSocket connections) and submit_grades (if LTI configured).

FSM Student Conversation FSM-001: In the student's conversation FSM, COLLAB_ENDED event fires. Student transitions from collaborative → app_embedded. end_collab action: show session summary, return to standard app view. Student's individual conversation continues from here.

FSM App Instance FSM-002: Shared app instance receives SESSION_END event → terminated. cleanup_instance action fires: destroy iframe, release resources, persist final state.

User Exit Paths

  • Engage with follow-up prompt: Student types in the chat input. The AI continues the individual conversation. The collab session is complete; only the individual chat continues.
  • Request a new session: Student asks the AI to start a new game/activity. AI invokes the app tool again, creating a new instance (FSM-002 starts fresh). A new collaborative session can be created.
  • Leave the platform: Student closes the tab or logs out. Session summary is preserved in conversation history. Next login shows the summary card in the chat thread.
  • Teacher views session data: After session ends, teacher sees an analytics entry for the collaborative session in the classroom's session list. Clicking shows the session summary and participant action counts.
  • Session summary not yet ready (loading state): If student's WebSocket receives the ending notification before the summary is ready, show an inline loading card: "Generating session summary..." This resolves when SUMMARY_READY fires.

Error Recovery

ERROR Summary generation fails (AI call fails): FSM error handling: default_error_state is ending, error_recovery_strategy is graceful_degradation. A fallback summary is generated using only the factual stats (no AI-written text): "Session lasted [X] minutes. [N] moves/actions taken." SUMMARY_READY fires with fallback content. Session still ends cleanly.

ERROR Session END POST fails (teacher action): Show inline error: "Could not end session. Try again." Session remains in active state. Teacher can retry. The session will eventually end via 30-min inactivity timeout if teacher cannot retry.

ERROR Student WebSocket closed before summary delivered: Summary is stored in the conversation history. When student reloads, GET /conversations/{id}/messages returns the summary card as a message in the thread. No data loss.

ERROR LTI grade submission fails: Logged as a warning (not critical). Teacher sees a "Grade submission pending" badge instead of the confirmation badge. A retry is scheduled. If all retries fail, the teacher is notified with instructions to manually submit the grade to the LMS.

ERROR Final state persist fails: This is a critical failure. The system retries 3 times with exponential backoff. If all fail, a system alert is sent to the platform admin. The session summary may be incomplete. A flag is set on the session record indicating a persist failure, visible to admin.

Accessibility Notes

  • Session summary card uses role="article" with aria-label="Collaborative session summary for [App Name]". Screen reader announces when it appears in the chat thread.
  • Stats row within the card uses a <dl> (description list): <dt> for stat name, <dd> for stat value. E.g., "Duration: 18 minutes."
  • LTI grade badge (if present) has aria-label="Grade submitted to [LMS name]". Not just a green checkmark — screen reader needs the text.
  • Follow-up AI prompt appears as a standard chat message. Chat input is re-enabled after session ends. Screen reader announces: "Chat input is now active. [AI prompt text]."
  • "Generating session summary" loading card has role="status": "Generating your session summary. Please wait." Announced once when it appears. Does not repeat while loading.
