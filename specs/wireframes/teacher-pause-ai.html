Teacher Pause AI

Screen: teacher-pause-ai | FR: FR-MON-004 | Persona: PER-002 (Ms. Torres the Teacher) | Journeys: JRN-003

Layout Zones

1
Confirmation Dialog (modal) — Student name and current topic. Pause action description: "The AI will stop responding to [Student Name]. They will see 'Your teacher has paused the AI — they'll be with you shortly.' You can send them a direct message while paused." Two primary buttons: "Pause AI" (destructive action, amber) and "Cancel". Optional: checkbox "Also send a direct message now" (expands inline input).
2
Student Context Summary (above buttons) — Last message the student sent (truncated to 2 lines), AI response status (responding / idle), active app indicator if any. Gives teacher confidence they are pausing the right student.
3
Resume Control (post-pause, replaces Pause button) — After pause is confirmed, the dashboard student card shows a "Paused" badge and a "Resume AI" button. This same control is surfaced inline on the student card and in the slide-out panel. Timer showing how long AI has been paused.

Trigger Conditions

TRIGGER From teacher-dashboard: Teacher clicks the "Pause AI" button on a student card's hover menu or from the slide-out conversation panel.

TRIGGER From safety-alert-detail: Teacher selects "Pause AI for this student" as an intervention action. Skips to confirmation dialog with alert context pre-filled.

TRIGGER Precondition check: If the student's AI is already paused, the button is replaced with "Resume AI" — this screen is not shown for an already-paused student.

API Endpoints

API Pause AI: POST /api/v1/conversations/{conversation_id}/pause — No request body required. Response: { "paused": true, "paused_at": "ISO8601" }. Side effects: sets ai_paused=true, sends WebSocket system_notification to the student with the pause message.

API Resume AI: POST /api/v1/conversations/{conversation_id}/resume — Called when teacher clicks "Resume AI". Response: { "paused": false }. Clears the pause notification from the student's chat.

API Optional direct message with pause: If teacher checks "Send a message too", the form includes a message field. On confirm, two calls fire sequentially: POST /pause then POST /conversations/{id}/teacher-message.

API WebSocket (student side): Student receives system_notification event on /ws/chat/{session_id}. No polling required.

State Transitions

FSM Teacher FSM-006: PAUSE_AI event from reviewing → intervened. log_intervention action records pause in audit trail. Teacher then follows RETURN_TO_MONITORING back to dashboard.

FSM Student FSM-001: TEACHER_PAUSE event on the student's conversation. Transition from active (or app_embedded) → paused. Entry action: show_pause_notification to student via WebSocket.

FSM Resume: TEACHER_RESUME event on FSM-001 → paused back to active. The resume_ai action calls POST /conversations/{id}/resume and removes the pause notification.

FSM Student behavior while paused: Student can still type in the chat input. Messages are stored but AI does not respond. Teacher messages can still be received. FSM-001 paused state allows TEACHER_MESSAGE events.

User Exit Paths

  • Confirm "Pause AI": POST fires → dialog closes → student card shows "Paused" badge → teacher returns to dashboard.
  • Cancel button: Dialog closes. No API call. Returns to invoking context (dashboard or alert detail).
  • Escape key: Same as Cancel.
  • Resume AI (post-pause, from dashboard card): POST /resume fires → "Paused" badge removed from student card → student's chat input re-activates.

Error Recovery

ERROR POST /pause fails (5xx): Show inline error in dialog: "Could not pause AI. Try again." Dialog stays open. Student is NOT notified of a failed pause.

ERROR POST /pause fails (404 — conversation ended): "This student's session has ended. No action needed." Dialog closes and card is refreshed to show offline status.

ERROR WebSocket disconnect (student side): If the student's WS is down, pause flag is set server-side. When WS reconnects, the client queries session state and renders pause notification retroactively.

ERROR POST /resume fails: Inline error on the student card's resume button. "Resume failed — try again." AI remains paused. Student remains in paused state.

Accessibility Notes

  • Confirmation dialog has role="alertdialog" (since it requires immediate attention) with aria-labelledby and aria-describedby.
  • Focus moves to dialog on open. Focus is trapped inside. On close, returns to trigger button.
  • "Pause AI" button uses aria-label="Pause AI for [Student Name]" to avoid ambiguity on the dashboard where multiple cards are present.
  • "Paused" badge on student card has aria-label="AI paused — [X] minutes ago".
  • Resume button on student card has aria-label="Resume AI for [Student Name]".
