Login (SSO)

Screen: login | Personas: All (PER-001 through PER-003) | Entry point for all user types

Information Hierarchy

1
SSO Buttons (center) — Three large, full-width buttons: Clever, ClassLink, Google. These are the primary action surface. Most users (students) will click Clever or ClassLink because their school provisions them.
2
Brand Identity (above buttons) — ChatBridge logo + tagline. Establishes trust and recognition. Brief — not a marketing pitch.
3
Teacher Email Login (below buttons) — Secondary path for teachers who don't use SSO. De-emphasized as text link because 90%+ of users authenticate via SSO.
4
Compliance Badge (footer) — COPPA/FERPA compliance indicator. Subtle but visible — builds trust for teachers and admins who are evaluating the platform.

Focal Point

FOCAL The SSO buttons own the first 3 seconds. The user should be able to identify their SSO provider and click within 2 seconds. The entire card fits on one screen with no scrolling at any viewport size.

Banned Dominance Patterns

  • No marketing content, feature lists, or testimonials on the login page
  • No hero image or illustration dominating the viewport
  • No username/password form as the primary path — SSO is the primary authentication method
  • No district selection dropdown — the SSO provider handles tenant routing

Interactive States

CLICK Clever button: Redirects to Clever OAuth flow. Shows loading spinner on button while redirect initiates. On Clever callback, routes to appropriate dashboard based on user role (student -> chat, teacher -> dashboard, admin -> catalog).

CLICK ClassLink button: Same flow as Clever but with ClassLink OAuth.

CLICK Google button: Google OAuth flow. Restricted to Google Workspace for Education accounts only — personal Gmail is rejected with error message.

CLICK Teacher email login: Expands to show email input + "Send magic link" button. No password — magic link only for security.

ERROR SSO failure: If SSO provider returns error, show inline error banner: "Clever login failed. Please try again or contact your school's IT department." No redirect to a separate error page.

HOVER SSO buttons: Subtle shadow elevation. Border color intensifies. Cursor: pointer.

API Integration Annotations

API Clever SSO: GET /api/v1/auth/clever/redirect — initiates OAuth flow. Callback: GET /api/v1/auth/clever/callback?code={code} — exchanges code for tokens, creates session, returns JWT + user role.

API ClassLink SSO: GET /api/v1/auth/classlink/redirect and GET /api/v1/auth/classlink/callback — same pattern as Clever.

API Google SSO: GET /api/v1/auth/google/redirect and GET /api/v1/auth/google/callback — validates domain is Workspace for Education.

API Magic link: POST /api/v1/auth/magic-link — body: { "email": "teacher@school.edu" }. Sends email with one-time login link.

API Role routing: After auth, GET /api/v1/auth/me returns user role. Frontend routes: student -> /chat, teacher -> /dashboard, admin -> /admin/catalog.

Responsive Behavior

MOBILE 375px Card goes full-width with 16px margin. SSO buttons remain full-width. Font sizes scale down slightly. The card is vertically centered and fits on screen without scrolling.

DESKTOP 1280px Card is 400px max-width, centered horizontally and vertically. Generous whitespace around the card.

Persona Fit

PERSONA This screen serves all user types but is optimized for students (primary users). Students on shared Chromebooks need to log in quickly at the start of a 30-minute class period — every second on the login screen is lost learning time. SSO buttons are large (44px+ touch target) and labeled with provider names students recognize from other school tools. The compliance badge is for teachers/admins evaluating the platform — students ignore it, which is correct.
