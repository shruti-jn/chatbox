Student Chat — Collaborative Session

Screen: student-chat--collaborative | Persona: PER-001 (Sam the Student) | Journey: JRN-002 (multi-student collaboration)

Information Hierarchy

1
Collaborative App Card (center) — The chess board with participant avatars and turn indicator is the focal point. This is the shared working surface. The turn indicator answers the immediate question: "Whose turn is it?"
2
Chat Thread (below/around app card) — Mixed messages from students and AI. Student messages are attributed (avatar + name). The chat is the coordination layer — students discuss strategy, AI provides coaching.
3
Participant Avatars (app card header) — Small circles with initials show who is in the session. Highlighted avatar indicates whose turn it is.
4
Move History (inside app card) — Algebraic notation of all moves. Latest move highlighted. This is reference data, not the primary focus.

Focal Point

FOCAL The chess board dominates the first 3 seconds — it is the largest visual element and the shared activity. The turn indicator banner immediately below the app card header directs attention to whose turn it is.

Above-the-fold: The app card (with chess board) and at least the last 1-2 chat messages visible without scrolling. The chat input is always pinned to the bottom.

Banned Dominance Patterns

  • No participant list sidebar consuming horizontal space — avatars are compact, in the card header
  • Move history must not be larger than the board itself — it is reference data
  • AI messages must not dominate the chat — collaborative sessions are student-to-student with AI coaching support
  • No separate "team chat" and "AI chat" tabs — all messages in one thread, differentiated by avatar

Interactive States

REAL-TIME Chess board: Updates in real-time when any participant makes a move. Move animation: piece slides from origin to destination (200ms ease-out). Last-moved squares are highlighted green.

REAL-TIME Turn indicator: Updates immediately when a move is made. Shows the next player's avatar and name. Subtle pulse animation on the active player's avatar.

REAL-TIME Chat messages: Messages from other students appear instantly via WebSocket. System messages ("Emma S. moved: c5") appear inline with a distinct gray style.

CLICK Chess board squares: On the active player's turn, clicking a piece highlights valid moves. Clicking a valid destination square makes the move. Invalid clicks are ignored.

CLICK Suggestion chips: Same as standard chat — clicking sends the chip text as a message. Chips are contextual to the game state ("Explain this position", "What should Emma play?").

HOVER Move history entries: Hovering a move highlights the corresponding squares on the board.

API Integration Annotations

API Session state: GET /api/v1/sessions/{session_id}/collaborative — returns participants, game state (FEN string), move history, whose turn it is.

API Real-time updates: WebSocket /ws/collaborative/{session_id} — pushes: move events, chat messages, participant join/leave, AI responses. All participants share one WebSocket channel.

API Make a move: POST /api/v1/sessions/{session_id}/app-action — body: { "app": "chess", "action": "move", "from": "c7", "to": "c5" }. Server validates legality. Broadcasts to all participants via WebSocket.

API Send chat message: POST /api/v1/sessions/{session_id}/messages — same as standard chat. Message is broadcast to all participants. AI may respond if the message is a question.

API AI coaching: AI automatically comments on interesting positions (triggered server-side by game state analysis). Pushed via WebSocket as an AI message. Rate-limited to prevent AI from dominating conversation.

Responsive Behavior

MOBILE 375px Chess board renders at full viewport width (375px - 32px padding = 343px). Move history collapses to a single line showing the last move only (expandable). Chat messages below the board. Collaborative sessions are challenging on mobile — but must work on Chromebooks at 768px+ viewport.

TABLET 768px Chess board at 360px with move history beside it. Chat thread below. This is the primary device (Chromebook).

DESKTOP 1280px Chat column centered at 800px max-width. Chess board 320px with move history beside it. Generous space for chat thread below.

Persona Fit

PERSONA Collaborative sessions are ChatBridge's signature differentiator. Students playing chess together with AI coaching in the same chat thread is the "magic moment" that sells the platform. The design must make collaboration feel natural: participant avatars establish presence, the turn indicator creates urgency, and student-to-student messages are first-class alongside AI messages. The AI acts as a coach, not a referee — it suggests and explains but does not interrupt the flow. System messages ("Emma moved: c5") provide game narration that keeps non-active players engaged during others' turns.
